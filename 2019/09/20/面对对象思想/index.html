面对对象思想
2019-09-20|OOP

概述

面对对象有四大特性：抽象，封装，继承，多态。面对对象思维，以对象模型为核心，丰富模型内涵，扩展模型外延，通过模型的行为组合去共同解决某一类问题。

  • 抽象是面对对象思想最基础的能力之一，即业务建模能力。
  • 封装是一种对象功能内聚的表现形式，使模块之间耦合度变低。
  • 继承是子类能够继承父类，获取父类的部分属性和行为，使模块更具有复用性。
  • 多态使模块在复用性的基础上更具有扩展性，使运行期更有想象空间。

抽象

抽象是程序员的核心素质之一，体现出程序员对业务的建模能力，以及对架构的宏观掌控力。抽象分为归纳和演绎。归纳的过程中，需要抽象出对象的属性和行为的共性。演绎是在已有问题解决方案基础上，正确找出合适的使用场景。

封装

封装是在抽象的基础上决定信息是否公开，以及公开等级。封装的主要任务是对属性，数据，部分内部行为实现隐藏。在不知道什么样的访问权限合适的时候，优先推荐使用private控制级别。下图是Java的访问权限：

继承

继承是面对对象技术的基础，允许创建具有逻辑等级结构的类体系，形成一个继承树，让软件在业务多变的客观条件下，某些基础模块可以直接复用，间接复用或者增强复用。符合里氏代换原则(LSP)，它指任何父类能够出现的地方，子类都能够出现。在设计中，提倡组合优先原则来扩展类的能力，而不是继承。

类关系：

  • 继承：extends(is-a)

  • 实现：implements(can-do)

  • 组合：类是成员变量(contains-a)。各个部分都有其独立的使用价值和生命周期，所有成员共同完成一件使命，生命周期一样，是一种非常强的整体和部分关系。

  • 聚合：类是成员变量(has-a)。是一种可以拆分的整体和部分的关系，是非常松散的关系。

  • 依赖：import类(use-a)。类只要import，就是依赖关系。

多态

多态是以上述三个面对对象特性为基础，根据运行时的实际对象类型，同一个方法产生不同的运行结果，使同一个行为具有不同的表现形式。具体表现形式有重写(overide)和重载(overload)。

  • 重写：是子类实现接口，或者继承父类时，保持方法签名完全相同，实现不同的方法体，是垂直方向的行为不同表现。Java中遵循一大两小两同。

    • 一大 : 子类的方法访问权限控制符只能相同或变大。

    • 两小: 抛出异常和返回值只能变小能够转型成父类对象。子类的返回值, 抛出异常类型必须与父类的

      返回值、抛出异常类型存在继承关系。

    • 两同: 方法名和参数必须完全相同。

  • 重载：是本一个类方法名称相同，但是参数类型或者参数个数不同，是水平方向行为的不同实现。JVM选择合适的目标方法顺序如下：

    1. 精确匹配。
    2. 如果是基本类型，自动转换为更大表示范围基本类型。
    3. 通过自动拆箱和装箱。
    4. 通过子类向上转型继承路线依次匹配
    5. 通过可变参数匹配。
