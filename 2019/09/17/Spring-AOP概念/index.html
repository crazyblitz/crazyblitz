Spring AOP概念
2019-09-17|Spring AOP

概念

横切的技术，是将那些影响了多个类的公共行为封装到一个可重用模块，并将其命名为Aspect(切面)。

使用切面解释，AOP把软件系统分为两个部分：核心关注点和横切关注点。业务处理的主要流程核心关注点，与之关系不大的部分是横切关注点。横切关注点的一个特点是，他们经常发生在核心关注点多处，而且各处基本类似。比如权限认证，日志，事务等。

使用场景

  1. 权限认证(Authentication)。例如场景的权限框架：Shiro和Spring Security。
  2. 缓存(Caching)。例如Spring提供的EnableCaching缓存注解。
  3. 错误处理(Error handling)。例如Spring提供RestControllerAdvice全局异常控制通知。
  4. 事务(Transactions)。例如Spring提供的事务注解Transactional。
  5. 同步(Synchronization)
  6. 持久化(Persistence)
  7. 记录(logging)，跟踪(tacing)，优化(profiling)，监控(monitor)。
  8. 校验(validation)
  9. 性能优化(Performance optimization)
  10. 懒加载(Lazy loading)
  11. 内容传递(Context passing)
  12. 资源池(Resource pooling)

核心概念

1：切面(Aspect)，Spring我觉得就是Advisor抽象。

切面是对横切关注点的抽象。在Spring可以理解为Advisor。Advisor是由一个Advice(通知)和一个Pointcut(切点)构成。常使用的Advisor如下：

  • StaticMethodMatcherPointcutAdvisor：Advisor静态基类，是静态切入点，方便子类实现MethodMatcher的matches方法自定义检查目标方法是否符合Advice条件，默认匹配所有方法。该类就是让子类实现的。最主要实现的方法是boolean matches(Method method,Class<?> targetClass);

  • DefaultPointcutAdvisor：便利的切点驱动Advisor实现。最常用的Advisor。它可以与任何切入点和建议类型一起使用，除了DefaultIntroductionAdvisor(引介切面)。

  • RegexpMethodPointcutAdvisor：jdk regex表达式Advisor。

  • NameMatchMethodPointcutAdvisor：存放建议的名称匹配方法切入点的便捷类。

2：切入点(Pointcut)

对连接点的定义。

  1. ClassFilter：过滤器限制切入点或引介与给定目标类集的匹配。
  2. MethodMatcher：过滤器限制切入点或引介与给定方法目标方法集的匹配。
  3. 横切关注点工具类集合
    • Pointcuts：提供切点的集合操作(集合交，集合并操作)。
    • ClassFilters：提供目标类过滤器的集合操作(同上)。
    • MethodMatchers：提供目标方法过滤器的集合操作(同上)。
  4. 实现类
    • StaticMethodMatcherPointcut：静态方法表达式切点
    • AspectJExpressionPointcut：AspectJ表达式切点
    • AnnotationMatchingPointcut：注解匹配切点
    • DynamicMethodMatcherPointcut：动态方法匹配切点
    • JdkRegexpMethodPointcut：jdk正则表达式切点
    • NameMatchMethodPointcut：替代JdkRegexpMethodPointcut。匹配方法*xxx,xxx*,*xxx*。

3：连接点(Joinpoint)

被拦截的点，由于Spring只支持方法级别的连接点，所以在Spring中连接点指的就是被拦截的方法。由于Spring还提供了引介增强，也可以是字段或者构造器。

  • JoinPoint：用来支持其他4种类型通知。
  • ProceedingJoinPoint：用来支持环绕通知(AroundAdvice)连接点。

4：通知(Advice)

指拦截到连接点之后要执行的代码，通知分为前置、后置、 异常、最终、环绕通知五类。

  1. 前置通知：方法返回前调用

    • BeforeAdvice：标记接口
    • MethodBeforeAdvice：前置通知
    • AspectJMethodBeforeAdvice

  2. 后置通知：方法返回后调用

    • AfterAdvice
    • AspectJAfterAdvice

  3. 环绕通知：方法前后调用

    • AspectJAroundAdvice

  4. 最终通知：返回后的建议仅在正常方法返回时调用，能看到返回值，但是不能改变。

    • AfterReturningAdvice
    • AspectJAfterReturningAdvice

  5. 异常通知：异常后调用

    • AspectJAfterThrowingAdvice

  6. 引介通知：可以在运行期间不改变原始类增加其他的方法。

总结Spring实现包装Aspect的5种通知实现

  • AspectJMethodBeforeAdvice
  • AspectJAroundAdvice
  • AspectJAfterAdvice
  • AspectJAfterReturningAdvice
  • AspectJAfterThrowingAdvice

目标对象

代理的原始对象。

织入

引入

在不修改代码的前提下，引入可以在运行期为类动态地添加一些方法或字段。在Spring中提供了

@DeclareParents注解来实现引介通知。

Spring AOP代理方式

Spring 提供了两种方式来生成代理对象: JDKProxy 和 Cglib，具体使用哪种方式生成由

AopProxyFactory 根据 AdvisedSupport 对象的配置来决定。默认的策略是如果目标类是接口，

JDK 动态接口代理

  1. JDK 动态代理主要涉及到 java.lang.reflect 包中的两个类：Proxy 和 InvocationHandler。

InvocationHandler是一个接口，通过实现该接口定义横切逻辑，并通过反射机制调用目标类

的代码，动态将横切逻辑和业务逻辑编制在一起。Proxy 利用 InvocationHandler 动态创建

一个符合某一接口的实例，生成目标类的代理对象。

CGLib 动态代理

  1. CGLib 全称为 Code Generation Library，是一个强大的高性能，高质量的代码生成类库，

可以在运行期扩展 Java 类与实现 Java 接口，CGLib 封装了 asm，可以再运行期动态生成新

的 class。和 JDK 动态代理相比较：JDK 创建代理有一个限制，就是只能为接口创建代理实例，

而对于没有通过接口定义业务方法的类，则可以通过 CGLib 创建动态代理。
