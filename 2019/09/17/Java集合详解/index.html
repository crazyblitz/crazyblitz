Java集合详解
2019-09-17|Java基础

概览

Java集合框架主要包括Collection和Map两种，Collection存储着对象的集合，而Map存储着键值对的映射表。框架图如下：

在集合框架图中，红色代表接口，蓝色代表抽象类，绿色代表并发包中的类，灰色代表早期线程安全类（基本已经启用）。从中可以看出，与Collection相关的4条设计线路分别是List,Queue,Set,Map。它们的子类会映射到数据结构中的顺序表，红黑树，哈希表，队列，栈等。

List集合

List集合是线性数据结构的主要实现。List集合的遍历结构是稳定的。该体系中最常用的是ArrayList和LinkedList两个集合类。

  • ArrayList：内部基于动态数组实现，支持随机访问，适合于查找，但是不适用删除和新增多的场景。
  • LinkedList：基于双向循环链表实现。适合于新增和删除多，不适用查找多的场景。该类实现了Deque接口，可以做队列，栈，双向队列的实现。测试表明，对于10万条数据，与ArrayList相比，随机提取元素时存在数百倍的差距。
  • Vector：和ArrayList类似，但是线程安全，早期的线程安全集合。建议在多线程时，如果不是数据很敏感，可以使用CopyOnWriteArrayList线程安全类，设计上使用了读写分离的思想。

Set集合

Set集合是不允许出现重复元素的集合类型。Set体系最常用的是HashSet,TreeSet和LinkedHashSet。

  • HashSet：基于HashMap实现，支持快速查找，但是不支持有序性操作。并且失去了元素插入顺序，使用

    Iterator遍历得到结果是不确定的。内部使用了Value固定为一个静态对象，使用Key保障集合元素的唯一性。

    LinkedHashSet：具有HashSet查找效率。继承自HashSet，内部使用链表维护了元素插入顺序。

  • TreeSet：基于红黑树实现，支持有序性操作。支持插入，查找，删除操作 时间复杂度log(n)。

Queue集合

Queue(队列)是一种先进先出的数据结构。主要实现是阻塞队列，优先队列，双端队列。

  • BlockingQueue：阻塞队列，主要支持生产者-消费者模式。支持take和put操作阻塞。
  • LinkedList：实现了Deque接口
  • PriorityQueue：基于堆结构实现，默认小根堆，可以用它实现优先队列。

Map集合

Map结合是以key-value键值对作为存储元素实现的哈希结构。key以某种哈希函数计算后是唯一的，Value则是可以重复的。可以使用keySet()查看所有的key,使用values()查看所有的value，使用entrySet()查看所有的键值对。

  • HashMap：基于哈希表实现。内部使用哈希链表法+红黑树(jdk8)。HashMap判断两个对象是否相等：依赖于对象的equals和hashCoded方法，判断代码如下：

  • TreeMap：基于红黑树实现，支持有序性操作。支持插入，查找，删除操作 时间复杂度log(n)。

  • LinkedHashMap：在HashMap的基础上，使用双向链表来维护元素顺序。顺序为插入顺序或者最近最少使用(LRU)顺序。

  • Hashtable：早期的线程安全map类。内部使用synchronized同步方法实现线程安全。

  • ConcurrentHashMap：线程安全的map。jdk7内部实现使用了分段锁，jdk8内部实现使用synchroinzed+cas实现。
