Java反射详解
2019-09-22|Java基础

前言

动态语言，是指程序在运行期间可以改变其结构：例如新的函数可以引入，已有的函数可以被删除等结构上的变化。比如常见的JavaScript，Ruby，Python等就是动态语言。而C，C++属于静态语言。从反射的角度说JAVA属于半动态语言。

反射

  1. 功能
  • 在运行期间，对于任意一个类可以知道该类所有的属性和方法。
  • 对于任意一个对象，都能调用它的任意一个方法或者设置属性。
  1. 优缺点
    • 优点：增加扩展性和重用性，调试器和测试功能。例如Spring的Bean的属性填充。
    • 缺点：性能开销大，安全限制，内部暴露。如果一个功能可以不用反射完成，最好不用。
  2. 应用场景：如果程序只能在运行时发现对象或者类的真实信息，就可以使用反射。

Java反射API

  1. 反射API主要由以下构成：最主要的由Class,Field,Method,Constructor四个类构成。即涵盖属性，方法，类，接口，参数，泛型，代理，数组，修饰符几大块。

    • Class类：反射的核心类，可以获取类的属性，方法，注解等信息。
    • Field类：代表类的属性。
    • Method类：代表类的方法。
    • Constructor类：代表类的构造方法。
    • AnnotatedElement类：代表注解。
    • Modifier类：代表属性或者方法的修饰符。
    • Proxy类：代表JDK代理。
    • ParameterizedType类：代表泛型参数类型。
    • Parameter类：代表方法参数。

  2. 反射使用步骤

    1. 获取想要操作的类的Class对象，可以通过Class.forName(className)。
    2. 调用Class类中方法，即反射的使用阶段。
    3. 使用反射API来操作这些信息。

  3. 获取Class对象方法

    • 调用某个对象的getClass()方法。
    • 调用某个类的class属性。
    • 使用Class类中forName()静态方法(最安全/性能最好)。

  4. 创建对象方法

    • 使用Class对象newInstance()，对象需要用默认构造函数。
    • 使用Constructor对象的newInstance()。需要先获取类的Constructor对象。

  5. Class类中场景方法解读

    • 带有Declared方法的参数：返回本类的所有修饰符的属性或者方法等，注意不包含继承的方法或者属性。示例如下：
      • getDeclaredFields
      • getDeclaredMethods
    • 不带有Declared方法参数：返回本类和继承父类的公有的属性或者方法。
      • getFields
      • getMethods
